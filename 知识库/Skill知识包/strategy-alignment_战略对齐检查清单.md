# 战略对齐检查清单

## 检查维度

### 1. 方向一致性
- 今天的决策和战略方向是否一致？
- 资源分配是否符合战略优先级？
- 是否有"忙次要事、漏重要事"的情况？

### 2. 原则一致性
- 用户表达的观点是否和之前写的原则矛盾？
- 是否有"说一套做一套"的情况？
- 是否违背了 CEO 工作原则中的任何一条？

### 3. 边界一致性
- 今天的工作是否超出了服务边界？
- 是否有应该拒绝但没拒绝的事项？
- 是否有应该升级给 CEO 但没升级的事项？

### 4. 优先级一致性
- 时间分配是否符合战略优先级？
- 是否把太多时间花在低优先级事项上？
- 是否有战略性事项被持续推迟？

## 输出模板

```markdown
# 战略对齐报告 —— YYYY-MM-DD

## 今日工作摘要
- 关键决策：...
- 关键观点：...
- 资源分配：...

## 对齐检查

### 方向一致性
- 状态：一致 / 冲突
- 详情：...

### 原则一致性
- 状态：一致 / 冲突
- 详情：...

### 边界一致性
- 状态：一致 / 冲突
- 详情：...

### 优先级一致性
- 状态：一致 / 冲突
- 详情：...

## 冲突详情（如有）
- 冲突点：你今天在 X 中表达了 Y
- 战略文档：在 Z 中写道...
- 建议：...

## 下一步
- 需要调整的事项：...
- 需要确认的问题：...
```

## 处理冲突的原则

- 直接指出，不回避
- 具体引用战略文档的原文
- 把冲突当作调整机会，不是指责
- 给用户选择权：调整行动 or 调整战略
