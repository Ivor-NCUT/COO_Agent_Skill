# Daily Review 工作流与日报模板

## 核心工作流

### Step 1：扫描当日工作上下文

读取以下信息源（根据用户实际配置选择可用渠道）：

- **飞书文档**：当天创建或编辑的文档
- **会议记录**：当天的会议记录、妙记
- **消息记录**：当天的 IM 消息（注意隐私边界）
- **待办任务**：任务列表状态
- **日历日程**：当天的会议和日程
- **Git 提交**：当天的代码提交记录
- **用户主动提供**：用户直接在对话中粘贴的材料

### Step 2：执行四类维护动作

#### A. 系统维护

更新到对应的管理表格：
- 项目排期表
- 任务协同表
- 客户/合作方管理表
- 内容/作品管理表
- 情报与机会表

更新对应文档：
- 会议记录合集
- 服务交付说明书
- 业务模型说明书

#### B. 知识沉淀

识别三种机会：
- 重复工作 → 写成 Skill/SOP
- 精彩洞察 → 做成选题
- Skill 迭代机会 → 记录 bad case

#### C. 待办清理

识别三类问题：
- 已承诺但未录入
- 已录入但长期未动
- 已过期但未更新

检查可自动完成项。

#### D. 战略对齐

对比当日工作与战略文档，识别逻辑冲突。

### Step 3：生成 COO 日报

## 日报模板

```markdown
# COO 日报 —— YYYY-MM-DD

## 今日扫描范围
- 文档 X 篇、会议 Y 场、消息 Z 条...

## 系统维护
- 已更新：...
- 待确认：...

## 知识沉淀建议
- 重复工作：...
- 精彩洞察：...
- Skill 迭代：...

## 待办清理
- 新增待办：...
- 过期提醒：...
- 可自动完成：...

## 战略对齐
- 一致/冲突：...
- 建议：...

## 下一步
- 需要你确认的事项：...
- 我会自动推进的事项：...
```

## 维护原则

- 先问后写
- 保留来源
- 不重复录入
