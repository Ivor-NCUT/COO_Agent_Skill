# 知识沉淀判断标准

## 重复工作识别

### 适合写成 Skill
- 未来会反复出现
- 需要 Agent 做判断、追问、诊断或生成方案
- 有明确触发场景
- 有稳定的输入、输出和流程
- 可以绑定知识包或案例库

### 不适合写成 Skill
- 只是单条知识
- 只是一次性总结
- 可以用简单脚本解决
- 没有可复用流程

### 适合做成 SOP
- 步骤确定
- 不需要主观判断
- 需要批量处理或定期运行

## 精彩洞察捕捉

### 适合做成选题
- 有传播潜力
- 符合用户人设
- 有具体场景支撑
- 属于流量/人设/转化三类之一

### 选题类型
- **流量选题**：公域热点、行业热议、名人洞察
- **人设选题**：来时路、理念价值观、交付过程、日常工作
- **转化选题**：客户误区、需求冲突、未被看见的痛苦、信息差观点

## Skill 迭代机会

### 记录 bad case
- 用户使用 Skill 后产出不满意
- 用户手动修改了 Skill 产出
- 用户在对话中多次调整同一类任务

### 触发迭代
- 用户明确说"帮我迭代这个 Skill"
- 用户分享了和 Skill 相关的外部经验
